Guidelines for Code Submission

To make the process more seamless and ensure users are aware of what to expect, here are some helpful instructions that could accompany the form:

  1. Naming Conventions
    • Please name your files in a clear and descriptive way. For example, if you are submitting a Python file for Step 1.4, name it step_1.4_solution.py.
  1. Code Comments
    • Include comments in your code explaining the logic behind major steps and functions. This helps the evaluator understand your approach.
  1. File Format
    • Ensure that your file is in the correct format (e.g., .zip if there are multiple files, or a specific extension for the language you're using like .py for Python).
  1. Testing Your Code
    • Before submitting, make sure your code runs correctly on your local machine. Any errors or issues found after submission may delay the evaluation.
  1. Evaluation Criteria
    • Code efficiency
    • Correctness of the solution
    • Code readability (proper comments, naming conventions, etc.)
    • Completion of the task as per instructions
  1. Feedback and Grading
    • After your code is evaluated, you will receive feedback within the expected evaluation time. Scores may range from 0 to 100, depending on your solution's quality and completeness.

Backend Considerations

If you're building this form on a website or application, here are a few technical details for handling submissions:

  • File Uploads: Ensure that your server can handle file uploads securely and store them safely, potentially limiting the file size.
  • Database: Create a record for each submission with the user’s details, submission date, and associated files. You can also track the evaluation status (pending, evaluated, etc.).
  • Email Notifications: Set up automated emails to notify users when their submission is received and when the evaluation is complete.
  • Security: Use validation and sanitation for all inputs to avoid security vulnerabilities (like file upload restrictions and proper input sanitization).
